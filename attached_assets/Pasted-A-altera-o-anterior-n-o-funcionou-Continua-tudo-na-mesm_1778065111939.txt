Regras adicionais obrigatórias
Além disso, no print/PDF tens de:

remover qualquer display: flex, grid, float, columns, position, transform, ou largura fixa que esteja a manter o gráfico à direita;

remover qualquer overflow: hidden ou largura limitada no contentor da tabela que esteja a cortar a última coluna;

garantir que a tabela tem width: 100% e que o contentor não a está a comprimir;

se o motor de PDF ignorar @media print, aplicar também a mesma classe/estilo diretamente no template HTML usado para exportação, ou adicionar uma classe tipo pdf-export / print-mode ao body durante a geração do PDF, porque alguns motores HTML-to-PDF ignoram ou duplicam mal os estilos de impressão.

Instrução crítica
Se o PDF estiver a ser gerado com uma biblioteca tipo html2pdf, html2canvas, ou semelhante, não assumes que @media print chega.
Nesse caso, aplica também os estilos numa classe dedicada ao modo de exportação, por exemplo:

css
body.pdf-export .l-down-layout {
  display: block !important;
  width: 100% !important;
}

body.pdf-export .table-wrapper {
  display: block !important;
  width: 100% !important;
  max-width: 100% !important;
}

body.pdf-export .chart-wrapper {
  display: block !important;
  width: 100% !important;
  max-width: 100% !important;
  margin-top: 16px !important;
  clear: both !important;
}
e ativa essa classe antes de gerar o PDF, removendo-a no fim, porque alguns fluxos HTML-to-PDF ignoram estilos só definidos em @media print.
